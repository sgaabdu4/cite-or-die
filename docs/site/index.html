cite-or-die

A small, inspectable, self-hosted reference implementation of legal-grade retrieval-augmented generation. Documents stay on your machine. Only the retrieved chunks for the current question cross to whichever model provider you configure. Every claim in the answer is verified verbatim against those chunks before the response is returned — or repaired and re-checked, or rejected.

The red dot is a recorded mouse cursor. Walkthrough: open Settings → pick a provider, paste a key (the server encrypts it before disk) → save → re-open settings to see only a fingerprint, never the key → upload a source → ask a question → click a citation chip to jump to the exact passage in the document. What this is

Tenant + matter isolation

Every chunk, vector, BM25 index, and citation graph is scoped by tenant and matter. Casbin ABAC + Presidio PII redaction at the walls.

Hybrid retrieval

Dense embeddings (hash or BGE-M3), BM25, and citation-graph PageRank are fused with reciprocal-rank fusion, then optionally reranked by a cross-encoder.

Verbatim citation check

Aho-Corasick over normalised text, with a Levenshtein fallback for whitespace and smart-quote drift. Answers without matching citations are repaired or rejected.

Append-only audit

Every upload, retrieve, generate, and runtime-config change is written into a SHA-256 hash-chained ledger. Tamper detection on read.

Swappable providers

FakeLLM, OpenAI, Anthropic, Ollama, and any OpenAI-compatible endpoint (DeepSeek, Kimi, Qwen, HF router). Switch from the UI, no env-var edit required.

Encrypted at rest

API keys are stored AES-256-GCM with an HKDF-SHA256 subkey derived per-tenant from auth_secret. The server never returns the key after it is written.

Run it locally (60 seconds, no key)

./install.sh
uv run cite-or-die serve --host 127.0.0.1 --port 8765

Open http://127.0.0.1:8765. The first run uses the deterministic fake provider, hash embeddings, and an in-memory vector store, so nothing leaves your laptop and you don't need an API key, a GPU, or Docker. When you want a real model, click Settings, pick a provider, paste a key, and save.

Threat model — what's protected

  • Document contents never leave the box unless the retrieval pipeline picks a chunk and the configured provider is hosted.
  • Provider API keys are written through the UI and stored encrypted under data/tenants/<tenant>/provider.enc (mode 0600). The HTTP API only returns a fingerprint (…cdef (sha256:1a2b3c4d)). No code path returns the plaintext key after it is written.
  • Tenant isolation: per-tenant HKDF subkey means one tenant's ciphertext cannot decrypt another's. Path-traversal in tenant_id is whitelisted at both the store and the API boundary.
  • auth_secret rotation invalidates every stored override (silent fallback to server defaults). Tamper or wrong key → returns "not configured" instead of crashing.
  • Audit: every set/delete of runtime config emits a runtime_config_changed hash-chained row with the fingerprint and never the key.

Production stack

docker compose up --build brings up Qdrant (vector store), Postgres (audit + chunks), Redis (cache), Caddy (auto-TLS), OpenTelemetry Collector, Prometheus, Loki, Tempo, and Grafana with dashboards pre-provisioned. PII-allowlist scrubbing runs in the collector before any export leaves the host.

Release gates

  • make eval-t2ragbench-100 — 100-row T2-RAGBench gate checks retrieval recall lift over BM25@1 and BM25@8 baselines.
  • make eval-graph — citation-graph PageRank lift gate.
  • make adversarial — generated PDF attacks (white text, zero-width, homoglyph) plus Garak/PyRIT-style prompt-injection probes. Weekly security workflow.
  • make mutation — 70% mutation-kill-rate gate via mutmut.
  • make release-security — pip-audit CVE scan + CycloneDX SBOM in dist/security/.
