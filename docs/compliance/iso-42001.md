# ISO 42001 Alignment

cite-or-die is designed for customer-controlled deployment.

- Governance: release artifacts include tests, eval gates, a dependency CVE scan, SBOM generation, and security/adversarial workflows.
- Data controls: documents remain inside the deployment boundary; audit payloads use allowlisted fields.
- Monitoring: OpenTelemetry, Prometheus, Loki, Tempo, and Grafana run in the compose stack.
- Risk management: prompt injection, retrieved-content injection, PII redaction, and citation faithfulness gates are tested.
- Change control: phase tags and release checks tie code, package version, and Docker image version together.

This document is an implementation alignment note, not a certification claim.
